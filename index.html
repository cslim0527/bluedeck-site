Features

One console for
your agent swarm

Stop juggling terminal windows. BlueDeck runs, watches and coordinates parallel agents from a single app.

Multi-terminal grid

Up to 16 real PTY sessions per workspace in an automatic grid layout. The same xterm.js WebGL renderer as VS Code — fast and accurate, just like your real shell.

Agent dispatch

Pick an agent from the pane-header dropdown to launch it in that terminal. claude · codex · gemini · aider are auto-detected.

Agent Hub (MCP)

An embedded MCP server registers with Claude Code on launch. Agents read each other's output and pass prompts back and forth for autonomous collaboration.

Broadcast mode

Send one prompt to every terminal in the current workspace at once — command the whole agent swarm in a single keystroke.

Git worktree isolation

The ⎇ button gives each terminal its own git worktree and branch, so parallel agents on the same repo never touch each other's files.

Agent status awareness

Every pane is tracked as working / needs-input / idle. When the app is in the background, macOS notifications ping you the moment an agent needs a hand.

Diff review & squash-merge

The ± button opens a changed-file list with a colored unified diff. One click squash-merges it back into the repo or discards it all — conflicts roll back safely.

Workspaces & session restore

Each workspace keeps its own terminal set and default directory. Reopen the app and the layout is restored — Claude Code panes resume the conversation with --continue.

How it works

Parallel development in three steps

1

Open a workspace, split panes

Add the repo you're working on as a workspace and the grid fills with terminals. Add as many panes as you need — up to 16.

2

Launch an agent in each pane

Pick Claude Code, Codex, Gemini or Aider from the dropdown. For trusted repos, ⚡ auto-approve runs them without a single prompt.

3

Command, review, merge

Send prompts from the prompt bar, watch progress on the status dots, review the results in the diff panel, then merge in one click.

Built on Rust

Light, fast and safe

A Tauri 2 + Rust core. It uses the OS WebView for roughly 1/10 the memory and a few-MB binary vs Electron. Data races and buffer overflows are blocked at compile time, with no GC.

Native performance

One PTY and reader thread per pane. Output streams over events so the UI never stutters, even under heavy logs.

Signed auto-update

The app checks for updates on launch and installs signed packages in a single click.

Korean IME & i18n

Smooth Hangul composition inside the terminal, with the system locale detected to auto-apply Korean or English.
